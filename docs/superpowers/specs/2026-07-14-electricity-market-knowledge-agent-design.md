# 电力市场知识库智能体设计

## 1. 目标与范围

创建一个在 Codex 对话中使用的专用智能体，面向电力市场初学者构建、查询、手动更新和导出可追溯的电力市场知识库。

首期覆盖：国家层面，以及江苏、浙江、山西、湖北、四川、山东、甘肃、安徽。正式入库内容仅来自政府部门、国家能源局及派出机构、地方政府及发展改革部门、交易中心等官方渠道。

## 2. 用户可执行的工作

- 用通俗语言解释基础概念，例如机制电价、容量电费、现货市场、中长期交易、绿电交易。
- 查询指定省份或交易品种的适用对象、管理要求、准入条件、参与流程与考核方式。
- 在用户明确下达指令后，检查全部或指定范围的新增、修订、废止和替代文件。
- 导出当前知识库为一个带日期的 Excel 工作簿。

## 3. 知识与文件模型

知识库包含四类可关联记录：

1. 基础概念：名称、通俗解释、关联机制、适用范围、来源文件、发文编号、链接、核验日期。
2. 政策文件：标题、发文编号、发布单位、发布日期、官方链接、适用地区、状态、首次入库与最后核验时间。
3. 省份规则：省份、交易品种、适用对象、管理要求、准入条件、参与流程、考核方式、对应政策文件和状态。
4. 更新记录：新增、更新、替代、废止、待核验等事件及时间和说明。

同一份政策文件可以支撑多个概念或规则。文件的修订、替代和废止关系独立保存；历史记录保留，不覆盖删除。

## 4. 更新与质量规则

更新仅由用户手动触发。智能体应先检索约定范围内的官方来源，再与既有政策文件比对，防止重复入库。

文件存在发文编号时必须记录。官方页面或文件未见正式文号时，使用“未见正式文号”，不得猜测或编造。链接失效、内容冲突、政策关系不明或证据不足的内容标为“待核验”，不得作为确定性规则输出。

每次更新后，智能体输出新增、更新、替代/废止、待核验与未发现变化的摘要。

## 5. 查询体验

智能体接受自然语言指令，并先提供适合初学者的简明解释，再给出结构化要点和来源。支持按概念、省份、交易品种、发文编号和状态查询。

示例：

- “解释机制电价，并给我来源和发文号。”
- “查看江苏省绿电交易的准入条件和参与流程。”
- “更新山西省电力现货市场相关政策。”
- “检查八省及国家层面自上次核验以来的新增政策。”

## 6. Excel 导出

每次导出创建一个带日期的 `.xlsx` 文件，至少含下列工作表：

| 工作表 | 内容 |
| --- | --- |
| 基础概念 | 概念、通俗解释、关联机制、适用范围、来源文件、发文编号、链接、核验日期 |
| 国家政策 | 国家层面政策和规则 |
| 江苏、浙江、山西、湖北、四川、山东、甘肃、安徽 | 每省独立；含交易品种、适用对象、管理要求、准入条件、参与流程、考核方式、来源文件、发文编号、链接、状态 |
| 更新记录 | 入库、更新、替代、废止、待核验等历史 |

导出应自动换行长内容，发文编号为空时展示“未见正式文号”。

## 7. 交付内容与验收

交付包含：Codex 专用智能体、结构化知识库模板、更新/查询/导出工具和一份结构正确的 Excel 示例文件。

验收覆盖：政策去重、文号记录、替代/废止关系、八省独立工作表、基础概念工作表，以及 Excel 可正常打开。
